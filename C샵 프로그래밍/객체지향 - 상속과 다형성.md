## 생성자 호출 순서
자식 클래스 객체 생성 시:
1. 부모 생성자 호출
2. 자식 생성자 호출

**이유**: 자식 객체는 부모의 기능을 포함하므로 부모를 먼저 초기화해야 함

## 형변환 (캐스팅)

### 업캐스팅 (Upcasting)
**개념**: 자식 타입 → 부모 타입으로 변환

**특징**:
- 암시적 변환 (자동)
- 항상 안전함

**문법**:
```csharp
부모클래스 변수 = 자식인스턴스;
```

### 다운캐스팅 (Downcasting)
**개념**: 부모 타입 → 자식 타입으로 변환

**특징**:
- 명시적 변환 (수동)
- 실패 가능 (런타임 에러 위험)

**문법**:
```csharp
자식클래스 변수 = (자식클래스)부모인스턴스;
```

## 다형성 구현

### 가상 함수 (Virtual Function)
**목적**: 부모 타입 참조로 자식의 재정의된 메서드 호출

**부모 클래스**:
```csharp
public virtual void Display() 
{
    // 기본 구현
}
```

### 오버라이딩 (Override)
**개념**: 부모의 가상 함수를 자식에서 재정의

**자식 클래스**:
```csharp
public override void Display() 
{
    // 재정의된 구현
}
```

**동작 원리**:
1. 업캐스팅된 상태에서도
2. 실제 객체 타입의 메서드가 호출됨
