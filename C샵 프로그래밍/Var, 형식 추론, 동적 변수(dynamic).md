## var

⦁	var을 사용하면 **입력되는 값**에 따라 **자동으로 형식이 결정**됨
⦁	C#의 var은 **프로그램 소스 코드 작성**의 **편의성**을 위해 만든 키워드
    즉, 모든 값을 다 담을 수 있는 키워드가 아님
⦁	모든 값을 담고 싶다면 **dynamic** 사용

## 형식 추론
⦁	C# 컴파일러는 **var로 선언된 변수**에 **저장되는 값**을 **자동으로 추론**하여 **적당한 형식**으로 **변환**되는 기능

## 동적 변수(dynamic)
⦁	사용 시 **하나의 변수**에 **자료형**이 **서로 다른 데이터값**을 **배정 가능**
⦁	필요에 따라 **또 다른 자료형**의 데이터로 **바꿀 수 있음**
⦁	**동적으로 선언한 변수**는 **프로그램의 실행 시점**까지 **자료형**에 대한 **검사를 하지 않음**
⦁	**개발자의 실수**로 인한 **오류 발생 가능**, but **다른 자료형**이 가지는 **제약**으로부터 **자유**
⦁	**지역 변수**, **전역 변수** 모두 사용 가능
⦁	**변수 선언 시점**에서 반드시 **값**을 **배정할 필요 없음**
⦁	**함수의 반환값**이나 **매개 변수**에 사용 가능
⦁	**null** 값 가질 수 있음
⦁	**권장하지는 않음** < **중요**
⦁	많이 사용 시 **프로그램 실행 속도 느려짐** + **리소스 많이 차지**함
⦁	**변수의 자료형**이 **프로그램 실행 시 결정** > 더 많은 **예외 상황** + **오류의 가능성**에 노출됨
